# 入出力

> **型**: I / O / D ／ **必須上流**: SPEC（依存辺 ✅）
> I＝系外アクタ発の入力・O＝系外アクタ宛の出力・D＝プロセス間の内部データ（系外へ出ない）。
> 依存方向：O→P（生成元）・O→ACTOR（受け手）・D→P（生成元）。

---

## I-1: ノードファイル群

<details><summary>⬡ I-1 · v0.1</summary>

```yaml
id: I-1
type: I
labels: []
scheduled: ""
edges:
  - to: SPEC-1
    ref_version: "0.3"
```
</details>

**もの**: Markdown＋YAMLフロントマター形式のノードファイル群
**発生源**: 仕様著者（ACTOR-1）が作成
**形式**: `.md` ファイル、YAMLフロントマター＋Markdown本文
**タイミング**: spec-inspector 起動時にファイルシステムから読み込む
