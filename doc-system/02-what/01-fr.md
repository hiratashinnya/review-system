# 機能要求

> **型**: FR ／ **必須上流**: SR（refines ✅）

---

## FR-1: ノード識別・参照整合性

<details><summary>⬡ FR-1 · v0.1</summary>

```yaml
id: FR-1
type: FR
labels: []
scheduled: ""
edges:
  - to: SR-1
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

各ノードに型・一意 ID・辺を持たせ、辺の参照先が実在する ID であることを保証する。

---

## FR-2: RULE 自動検査

<details><summary>⬡ FR-2 · v0.1</summary>

```yaml
id: FR-2
type: FR
labels: []
scheduled: ""
suppress: [RULE-018]  # エラーパスなし: RULE 検査自体は fail-close で中断せず全件列挙する設計
edges:
  - to: SR-2
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

ノードセットを受け取った spec-inspector が、全 RULE に対する違反を G# 番号付きで列挙できる。

---

## FR-3: suppress 機構

<details><summary>⬡ FR-3 · v0.1</summary>

```yaml
id: FR-3
type: FR
labels: []
scheduled: ""
edges:
  - to: SR-1
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

ノードが理由付きの suppress リストを持つとき、指定した RULE をそのノードに対してサイレントにできる。

---

## FR-4: scheduled 属性によるフェーズ制御

<details><summary>⬡ FR-4 · v0.1</summary>

```yaml
id: FR-4
type: FR
labels: []
scheduled: ""
suppress: [RULE-018]  # エラーパスなし: scheduled は単純フラグで fail する状態を持たない
edges:
  - to: SR-1
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

ノードに scheduled 属性を設定すると、そのノードに対する全 RULE がサイレントになる。

---

## FR-5: ref_version による辺のバージョン固定

<details><summary>⬡ FR-5 · v0.1</summary>

```yaml
id: FR-5
type: FR
labels: []
scheduled: ""
suppress: [RULE-018]  # エラーパスなし: バージョンミスマッチは警告（RULE-003/004）として扱い処理継続
edges:
  - to: SR-3
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

辺の ref_version に参照先ノードのバージョンを固定し、バージョンが変化したときに検出できる。

---

## FR-6: 型別テンプレート提供

<details><summary>⬡ FR-6 · v0.1</summary>

```yaml
id: FR-6
type: FR
labels: []
scheduled: ""
suppress: [RULE-018]  # エラーパスなし: テンプレート取得は参照のみで失敗ケースなし
edges:
  - to: SR-1
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

各ノード型のテンプレートファイルを提供し、著者が必須フィールドと本文フォーマットを確認できる。

---

## FR-7: 工程別スキルへの著作規約統合

<details><summary>⬡ FR-7 · v0.1</summary>

```yaml
id: FR-7
type: FR
labels: []
scheduled: ""
suppress: [RULE-018]  # エラーパスなし: スキルは参照ドキュメントであり実行時に失敗しない
edges:
  - to: SR-1
    kind: refines
    status: pending
    ref_version: "0.1"
```
</details>

工程別スキルにノード著作規約（型・ID・辺・本文・RULE）を内包し、外部ファイル参照なしに著作できる。
