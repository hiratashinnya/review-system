## FR-11 改訂（v0.3 → v0.4）

<details><summary>⬡ FR-11 · v0.4</summary>

```yaml
id: FR-11
type: FR
labels: []
scheduled: ""
suppress: []
edges:
  - to: SR-1
    ref_version: "0.2"
```
</details>

型別テンプレート（`templates/<layer>/<type>.md`）を提供し、著者がテンプレートを複製した時点で id・type・labels・scheduled・edges（ref_version 付き辺エントリ）と型別本文フォーマット（前提条件・入力/トリガ・期待動作・例）の必須フィールドが揃った状態を保証する。テンプレートを複製するだけで RULE-025（id 欠如）・RULE-026（type 欠如）・RULE-027（ref_version 欠如）が発火しない初期状態を提供する。

## FR-13 新規（v0.1）

<details><summary>⬡ FR-13 · v0.1</summary>

```yaml
id: FR-13
type: FR
labels: []
scheduled: ""
suppress: []
edges:
  - to: SR-1
    ref_version: "0.2"
```
</details>

型別著作エージェント（`requirements-author`・`spec-author`・`analysis-author`・`design-author`・`verification-author`）に著作規約（型・ID PREFIX・辺方向・本文フォーマット・RULE チェックリスト）を内包させ、外部ファイル参照なしに正しくノードを著作できるようにする。また各層の著作ワークフロー（著作エージェント呼び出し → `tmp/<sprint>/<id>.md` に出力 → reconciliation → 本ファイル反映）を定義し、著作フロー全体の整合性を確保する。

## FR-14 新規（v0.1）

<details><summary>⬡ FR-14 · v0.1</summary>

```yaml
id: FR-14
type: FR
labels: [post-mvp]
scheduled: "post-mvp"
suppress: [RULE-018]  # 異常系なし: post-MVP の支援表示機能であり、ドリフト検出（FR-4）の失敗系は FR-4 側で担保される
edges:
  - to: SR-6
    ref_version: "0.2"
```
</details>

上流ノードが更新されてドリフトが検出されたとき（FR-4・RULE-004）、下流ノードが持つ ref_version 更新が必要な辺の一覧を表示し、著者が反映漏れを効率的に解消できる支援機能を提供する（post-MVP）。
