# 02. 評価実行と指摘の仕分け

AI が生成した指摘を、2つの軸で自動的に仕分けする。これがツールの肝。

## 軸A：深刻度（Severity）

- **Error** — 必須修正
- **Warning** — 推奨
- **Info** — 参考

## 軸B：対応モード（誰が直すか）

- **Auto-fix** — AI が修正案まで生成し適用（例：命名規則、誤字、フォーマット）
- **Suggest** — AI が提案、人間が判断（例：設計の是非、表現の妥当性）
- **Human-only** — AI は指摘＋原案提示まで、適用は人間（例：仕様の意思決定）

## A × B マトリクスで分類

各指摘を A×B に分類することで、「自動完結できるもの」と
「人間が要るもの」がきれいに分かれる。

> 重要：**Human-only でも AI は黙らない。** 原案・複数案を必ず提示し、
> 人間の仕事を「ゼロから考える」から「提示された案を判断する」に変える。

## 仕分けの根拠

仕分けは AI の裁量任せにせず、**基準ファイル側のメタ情報**で決める。
（`determinism` / `auto_fix` 等。詳細は 01）

AI に「これは決定論的か？」を毎回判断させると揺れるため、
基準を書く時点で人間が宣言しておく。

## ユーザーへの提示（3区分＋未分類）

```
評価レポート
├─ 🤖 自動修正済み（N件）  ← 承認不要で適用済み・流し読み用（デフォルト折りたたみ）
├─ ✋ 要承認（M件）        ← diff 提示・人間が適用ボタン
├─ 💬 要判断（K件）        ← 原案あり・人間が決定
└─ ❓ 未分類（L件）        ← id 未確定。自動仕分け対象外・新ルール候補（Q7）
```

未分類は LLM が既存 `rule id` に紐づけられなかった指摘。捨てずに surfacing し、育成（[04](04-feedback-loop.md)）の素材にする。

具体的な自動化レベルの線引きは [03-auto-fix-policy.md](03-auto-fix-policy.md) を参照。
