# 04. フィードバックループ（基準育成）

## 狙い

評価基準を運用しながら育てる。人間の判断がデータになり、AI が賢くなる。

```
初期設定（AI が叩き台を生成）
  ↓
AI が評価 → レビュアーが結果を承認 / 修正
  ↓
修正・違和感を基準 / ポリシーに反映（PR 化）
  ↓
管理者が承認
  ↓
次回から振る舞いが変わる（精度アップ）
```

## 流し読みが入口になる

「自動修正サマリを人間が流し読みする」行為そのものがフィードバックの入口。

```
自動修正サマリを流し読み
  ↓
違和感のある修正に「対象外」フラグ
  ↓
基準ファイルの auto_fix 設定変更を提案（PR 化）
  ↓
承認されれば次回から振る舞いが変わる
```

→ **人間の流し読みが、そのまま自動化レベルのチューニングデータになる。**
基準育成と自動化ポリシーがここで一本につながる。

## 必要機能

| フェーズ | 機能 |
|---|---|
| 初期 | 基準のひな形を AI が生成 |
| 運用 | AI の評価結果に人がフィードバックできる |
| 育成 | フィードバックを基準に取り込む（PR / 管理画面） |
| 可視化 | 基準の変遷履歴が見える |

## ガバナンス上の論点（未決）
