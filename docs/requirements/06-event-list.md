# 06. イベントリスト（event-response 表）

外部の出来事（トリガ）ごとに「**外部トリガ → システムの反応 → 出力 → 生む価値**」を1行で並べた表。
目的は機能設計ではなく、[05 I/O 台帳](05-io-overview.md)の `I-#/O-#` と突き合わせて
**どの入出力がどのイベントから駆動されるか**を可視化し、**考慮漏れ（孤児・穴）を炙り出す**こと。

凡例：種別＝実（外部アクターの行為）/ 時（時間・蓄積）/ 例外（異常系）。関連は `I-#`/`O-#`/`Q#`。

> **アクター列は記述的な呼称**。MVP はアクターを区別する仕組みを持たない（単一ユーザー・Git ホスト権限にも乗せない）。
> ~~E6/E7（メンテナ提案→管理者承認）~~ は**削除**：基準/ポリシー編集は**系外（システムを介さない・非イベント）**。
> 検査は合成時に毎回行い警告するだけでよい（[Q15](../dashboard.md)）。残るアクター論点（誰が有効化するか）は将来（[Q1](../dashboard.md)）。

## 新たに浮いた論点（→ dashboard へ Q 起票）

- **Q14 同一文書の再評価時の挙動** — 修正→再提出で前回との差分提示・重複指摘/重複自動修正の抑制。
  Q9（承認打ち上げ側の dedup）とは別レイヤ（**レビュー側の dedup**）。
- **Q15 基準/ポリシー更新の伝播** — マージ後の再合成トリガ、矛盾チェックのキャッシュ無効化、
  過去にレビュー済みの文書へ遡及再評価するか否か。
- ~~**Q16 承認待ち滞留への督促**~~ — **クローズ（削除）**：承認待ちが片付いたかを**システムは観測できない**（ステートレス・対応は系外）ため督促の前提が無い。E13/O-13 削除。
- **Q17 異常系イベントの振る舞い** — 基準パース失敗・LLM 障害・スコープ未解決・空文書での
  フェイルクローズ/オープン方針とエラー通知。出力 O-14（post-MVP）。

入出力の異動：~~O-13~~ 削除・**O-14**（異常系・post-MVP）・**I-12**（時間トリガ＝E10 のみ）・~~I-11~~ 削除（scope は I-4 内）。
→ [05 I/O 台帳](05-io-overview.md) に反映済み。
