# プロセス設計 03 — 状態インベントリ（どこに状態が要るか）

DFD のデータストア＝**状態が必要な場所**。ここを明示して「ステートレスでよい所」と分ける。

## 状態を持つ／持たないの判断基準

これまでの決定（[Q14](../dashboard.md) ステートレス、[Q9](../dashboard.md) 警告レジャー、[Q15](../dashboard.md) 毎回合成）から導いた基準：

- **毎回作り直せるもの → 状態化しない**：観点パック・メタ表（毎回合成）、findings、レポート、未分類。
- **過去を覚えていないと成立しないもの → 状態化する**：警告の既出、適用の取り消し可能性、フィードバック蓄積。

## データストア一覧（＝必要な状態）

| ID | ストア | 何の状態か | なぜ状態が要るか | 永続性 | MVP |
|---|---|---|---|---|---|
| DS1 | 基準/ポリシーファイル | 観点・メタ・ポリシーの定義 | システムの判断の源泉。外部（Maintainer）が編集する | 永続 | ○ |
| DS2 | 矛盾チェックキャッシュ | `content_hash → 本文矛盾判定結果` | **最適化のみ**（無くても再計算可）。高価な PF 判定の再実行回避（[Q15](../dashboard.md)） | 半永続（捨ててよい） | ○ |
| DS3 | finding-commit ワークスペース | 自動適用を finding 単位コミットで保持 | **revert を成立させるため**（過去の適用を覚えていないと戻せない・[Q3](../dashboard.md)） | 永続（実行をまたぐ） | ○ |
| DS4 | 警告レジャー | `{rule_id, content_hash, first_seen}` | **既出警告を出さないため**（"一度示せば足りる"・[Q9](../dashboard.md)） | 永続 | ○ |
| DS5 | フィードバック蓄積 | 却下・対象外・傾向 | 観点FB提案(O-12)の素材。過去の傾向が要る | 永続 | △（MVP 最小） |

## ステートレスで通すもの（状態化しない）

| 対象 | 理由 |
|---|---|
| 観点パック・メタ表 | 毎回合成（[Q15](../dashboard.md)）。導出物なので保持しない（DS2 は判定の最適化キャッシュのみ） |
| findings / unmatched | 1 実行の産物 |
| 評価レポート | 1 実行の産物（再提出すれば再生成） |
| レビュー対象どうしの dedup | 持たない（[Q14](../dashboard.md)）。問題が残る限り毎回出す |

## MVP の状態最小セット

DS1（基準ファイル）／DS3（revert 用 git）／DS4（警告レジャー）が**必須**。
DS2 は最適化なので**後回し可**（毎回 PF 判定でも動く）。DS5 は育成機能と一緒に最小から。
