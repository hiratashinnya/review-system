検証層・意思決定ノード（TD / TC / TR / VERIFY / FND / DD / Q / PEND）の著作支援プロンプト。外部ファイル参照なしに v2 slug id・path 由来 type/status・必須辺方向・追加属性・本文フォーマット・RULE チェックリストを内包する（SPEC-27 の実体＝`.claude/agents/verification-author.md`）。
**バージョン**: 1.0
**目的**: 指定された親ノード配下の検証・意思決定ノードを、v2 の slug id、path 由来 type/status、必須依存辺方向（TD→SPEC・TC→TD・TR→TC・VERIFY→any・FND は状態で逆転・DD/Q/PEND→影響要素の義務辺）・追加属性（TD の condition＝SPEC と一致＋`body_ref.*`／TC の `test.*`／TR の result・log_ref）・RULE チェックリスト（RULE-016/019/020/021/001/002/022 とライフサイクル）を**プロンプト内に閉じて**提供する（SPEC-27-1〜5）。
**入力変数**: `parent_id`／`parent_body`／`sprint`／`context`／`error`。記載内容（I-9）＝各型本文＋メタ属性（TD＝condition＋共有本文参照、TC＝実テスト識別子、TR＝result/log_ref、FND＝path status＋指摘時 ref_version の本文記録＝DD-3、DD/Q/PEND＝path status）。
**出力形式**: `tmp/<sprint>/<parent-id>/nodes/...` に corpus ミラーレイアウトで `{slug}.yaml` を必ず置く。required 型は同名 `.md`、TD は `body_ref.file`/`body_ref.anchor` で共有本文を参照、TC は Markdown 本文なし。本ファイルへは書かない。
**注意事項**: 辺は無名依存辺（kind/status なし・to は単数・ref_version 必須）。TD-TC は 1:1。FND は状態で辺逆転（未解消＝forward `FND→対象`／resolved＝backward `対象→FND`＋forward 削除＋本文に指摘時 ref_version を凍結＝DD-3）。DD/Q/PEND の義務辺は反映後 `DD→X` を削除し `X→DD` に置換。接続規則変更を伴う DD/FND は対応 author/スキル/接続マトリクスへ同期（FND-99 パターン）。`scheduled` は非空必須。現行スプリントは `scheduled: "sprint-1"` などで明示し、後送りはオーナー承認済み sprint 値を指定。`suppress` / `suppress_reason` は issue #118 で廃止済みのため出力しない。RULE-007 は always_error として常時 ERROR 発火する。
