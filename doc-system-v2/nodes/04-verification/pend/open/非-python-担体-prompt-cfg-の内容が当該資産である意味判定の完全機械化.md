**保留事項**: DD-10 の `src_symbol_eligibility` で `prompt=[file]`・`cfg=[file]` とした通り、prompt/cfg は Python ソースではなく **非 Python 担体（.md / .yml 等のファイル）**である。sprint-1 の file 適格性は「**存在＋正規パス規約一致**」（skill/agent なら `.claude/skills/<name>/SKILL.md` 等の固定パス、cfg なら規定の設定ファイルパス）までを機械判定する。しかし「そのファイルの**中身が当該資産である**」こと——例: SKILL.md が本当に当該スキルのプロンプト仕様を記述しているか、.yml が当該 cfg スキーマに沿う設定か——の**意味判定の完全機械化**は sprint-1 では行わない。この完全機械化を本 PEND として保留する。

**現状（sprint-1 の到達点）**: file 適格性＝存在＋正規パス規約一致まで。パスが規約どおりに在れば適格とみなし、内容の意味的な一致（当該資産であることの実質判定）は人手確認に委ねる（PR2 の運用ルール側）。

**影響要素**: `接続ルールスキーマ`（`src_symbol_eligibility` の file 判定条件を「存在＋パス規約」から「内容の意味一致まで」へ拡張する対象）。起源＝DD-10。

**実施時期はオーナー判断**: 本 PEND の `scheduled` は起票時点＝`sprint-1`。実施 sprint を独断で未来へ繰り越さない（CLAUDE.md「スケジュール独断禁止」）。中身の意味判定完全機械化を今スプリントで着手するか後続へ回すかはオーナーが決める。
