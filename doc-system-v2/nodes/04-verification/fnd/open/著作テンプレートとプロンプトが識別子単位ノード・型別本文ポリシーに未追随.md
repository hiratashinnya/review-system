**内容**: DD「識別子単位ノードは1ノード1YAMLを維持し本文は型別ポリシーで省略・共有を許可する」により、TD/TC/SRC の著作単位と本文方針が変わるが、現行テンプレート・test-strategy skill PROMPT・verification-author PROMPT は旧前提を含む。

**確認した事実**:

- `templates/verification/tc.body.md` は TC body に実装ファイルとテスト関数を書く前提である。
- `templates/verification/td.body.md` は1 TD body を単独 node body として扱う前提であり、shared body の `body_ref` を持たない。
- `test-strategy` PROMPT は TD/TC/TR 3層の出力形式を持つが、TD shared body と TC bodyless をまだ案内しない。
- `verification-author` PROMPT は TD/TC/TR/VERIFY/FND/DD/Q/PEND の著作支援を担うため、型別本文ポリシーに追随しないと新規著作が旧形式へ戻る。

**期待される解消**:

1. TD/TC/SRC テンプレートを型別本文ポリシーへ更新する。
2. test-strategy skill PROMPT を「1テストプログラム1 TD shared body、TD-TC 1:1、TC bodyless」へ更新する。
3. verification-author PROMPT を `body_ref` / `test` / `source` 属性に対応させる。
4. author/reconciliation validator 系プロンプトが `.md + .yaml` ペア固定を要求しないようにする。

**依存**: FORMAT/schema/validator の正本が決まった後に実施する。先にプロンプトだけ変更すると、著作結果を現行 validator が受け取れない。
