**廃止済み（issue #118・suppress機構の廃止に伴い本アサーションは対象消滅）**

**前提条件**: suppress 機構が動作しており、存在しない ID 参照を含むノードがある
**入力/トリガ**: ノードの `suppress` リストに RULE-007 が含まれる
**期待動作**: suppress に RULE-007 が含まれるとき、RULE-007 の ERROR を報告する（always_error のため抑制不可）。

本アサーションが検証対象としていた `suppress` 機構自体が issue #118 でオーナー方針により廃止された（RULE-004 等のドリフト判定は凍結免除せず常に発火させる設計に一本化）。検証対象が消滅したため、本 SPEC は退役する。
