**前提条件**: in-graph にノードが1件以上存在し、各ノードの summary バッジが走査済みである。
**入力/トリガ**: 検証ツールが全ノードの summary バッジから `· vX.Y.Z` 部分を抽出する。
**期待動作**: 全 in-graph ノードのバッジを走査したとき、各バッジが `· vX.Y.Z`（X・Y・Z はそれぞれ非負整数）の 3 部形式を持つことを判定する。z は ref_version 伝播判定に不問だがバッジ形式として必須。
**合格例**: `⬡ FR-1 · v0.3.0` → バッジ version = "0.3.0"（X=0, Y=3, Z=0）・形式適合 → 違反なし。`⬡ SPEC-54 · v0.1.0` → 違反なし。
