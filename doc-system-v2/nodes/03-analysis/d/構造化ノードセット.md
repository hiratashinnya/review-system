**もの**: P-1 が I-1（ノードファイル群）をパース・スキーマ検証して生成した構造化ノードの正規化集合。各ノードが id / type / labels / scheduled / edges（to・ref_version）の共通フィールド、および型別拡張フィールド（SPEC/TD の condition・TR の result / log_ref・suppressed ノードの suppress / scheduled スライス）を内部表現として含む
**種別**: 内部データ（系外へ出ない）・**P-1 内部の中間生成物**
**生成元**: P-1（ノードパース）に依存（D→P）
**消費先**: **P-1 内部のみ**（P-1-5→P-1-6 間の内部受け渡し）。外部プロセスへの配布は D-17〜D-22 の消費スライスが担う（スタンプ結合解消）

**改訂理由（MINOR バンプ v0.1→v0.2）**:
- 旧定義は「id/type/labels/scheduled/edges のみ」とあり condition / result / log_ref が D-4 に含まれないとも読めた。これにより P-2-3（condition 属性検査）・P-2-4（result/log_ref 検査）・P-3-2（仕様カバレッジ）の入力が台帳上断絶していた（PR6 価値経路の穴）。本改訂で型別拡張フィールドを内部表現に含むと明記し、断絶を解消する。
- 消費先列を「P-1 内部のみ」に改訂。従来の「P-2/P-3 系が消費」はスタンプ結合（全ノード全属性を使わない消費先へ丸渡し）。設計で D-17〜D-22 の消費スライスに分割したため、外部消費は各スライス D が担う。
