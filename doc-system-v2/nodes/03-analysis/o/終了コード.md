**もの**: spec-inspector 実行の終了コード（ERROR ≥ 1 件のとき 1・ERROR 0 件のとき 0）。CI ゲート信号として自動パイプラインが違反有無をコードで判定する。
**生成元**: P-4-4（終了コード決定）に依存（O→P）——終了コードの実生成リーフは P-4-4 であり、リーフ先例（O-3→P-7-2）に倣って生成元辺を終端リーフ P-4-4 に張る（親 P-4 ではない）。
**受け手**: レビュアー（ACTOR-2）に依存（O→ACTOR）——CI/自動パイプラインが終了コードを評価し、レビュアーが結果を確認・承認する。
**形式**: プロセス終了コード整数（0 または 1）。stdout ではなく OS プロセス終了コードとして返される。
**根拠仕様**: SPEC-25-2（ERROR≥1→終了コード1）・SPEC-25-3（ERROR=0→終了コード0）
