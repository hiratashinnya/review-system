P-2-1〜P-2-4 の各検査子から受け取った全違反候補に対し、scheduled（フェーズ限定発火）・activate_stage（ステージ限定発火）・always_error（常時 ERROR 発火）の規則を適用して、発火確定済み RULE 違反リスト（D-6）を確定する。発火条件と重症度確定を一元管理することが単一責務であり、各検査子は「論理違反の検出」のみに集中できる。
**責務（単一動詞）**: 発火規則と重症度規則を違反候補に適用して D-6 を確定する
**提供価値**: scheduled/activate_stage/always_error の適用が P-2-1〜P-2-4 に重複コピーされる横断関心事を 1 プロセスに集約し、発火規則の変更箇所を単一化できる
**入力**:
- 各検査子（P-2-1-1/1-2/2-1〜4/3-1〜4/4-1〜3）から送出される違反候補（P-2 親経由）
- D-9（フェーズ・ステージ状態・current_phase/current_stage/phases/stages・P-5-3 が生成）を消費（P→D）
- D-12（always-error 規則・scheduled/ステージ制御に関わらず常時 ERROR 発火させる RULE 群・P-5-3 が生成）を消費（P→D）
- D-14（ルール発火ステージ表・rule_activation・P-5-3 が生成）を消費（P→D）
- D-18（属性検査ビュー・scheduled スライスを含む・P-1-6 が生成）を消費（P→D）
**出力**: D-6（RULE 違反リスト・発火確定済み）が P-2-5 に依存（D→P）、P-4 が消費
**トリガ**: E-1 に依存（P→E）
